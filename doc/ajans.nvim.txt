1. ajans.nvim                                 *ajans.nvim-ajans.nvim*

**ajans.nvim** is your Neovim AI ajans for working with AI CLI tools without
leaving your editor. Chat with assistants, send rich buffer context, and keep
sessions alive inside your normal Neovim workflow.

FEATURES                                *ajans.nvim-ajans.nvim-features*

- Integrated AI CLI terminal for Claude, Gemini, Grok, Codex, Copilot CLI,
  and other tools.
- Context-aware prompts for buffers, files, selections, cursor positions,
  diagnostics, quickfix entries, functions, and classes.
- Prompt library for common tasks such as explaining code, fixing issues,
  reviews, tests, and documentation.
- Session persistence with tmux.
- Automatic file watching for changes made by AI tools.
- Picker actions and statusline helpers for plugin integration.

REQUIREMENTS                        *ajans.nvim-ajans.nvim-requirements*

- Neovim >= 0.11.2
- snacks.nvim for better prompt/tool selection (optional)
- nvim-treesitter-textobjects main branch for `{function}` and `{class}`
  context variables (optional)
- tmux for persistent CLI sessions
- AI CLI tools such as Codex, Claude, Copilot, Gemini, etc. (optional)
- lsof and ps on Unix-like systems for running process and port detection
  (optional, but recommended)

QUICK START                          *ajans.nvim-ajans.nvim-quick-start*

1. Install the plugin with your package manager.
2. Run `:checkhealth ajans`.
3. Install at least one AI CLI tool.
4. Use `<leader>aa` to open your current CLI session, `<leader>as` to select a
   tool, and `<leader>at` / `<leader>af` / `<leader>av` to send context.

INSTALLATION                        *ajans.nvim-ajans.nvim-installation*

Example lazy.nvim setup:

>lua
    {
      "assagman/ajans.nvim",
      opts = {
        cli = {
          mux = {
            enabled = true,
          },
        },
      },
      keys = {
        { "<c-.>", function() require("ajans.cli").focus() end, mode = { "n", "t", "i", "x" }, desc = "Ajans Focus" },
        { "<leader>aa", function() require("ajans.cli").toggle() end, desc = "Ajans Toggle CLI" },
        { "<leader>as", function() require("ajans.cli").select() end, desc = "Select CLI" },
        { "<leader>ad", function() require("ajans.cli").close() end, desc = "Detach a CLI Session" },
        { "<leader>at", function() require("ajans.cli").send({ msg = "{this}" }) end, mode = { "x", "n" }, desc = "Send This" },
        { "<leader>af", function() require("ajans.cli").send({ msg = "{file}" }) end, desc = "Send File" },
        { "<leader>av", function() require("ajans.cli").send({ msg = "{selection}" }) end, mode = { "x" }, desc = "Send Visual Selection" },
        { "<leader>ap", function() require("ajans.cli").prompt() end, mode = { "n", "x" }, desc = "Ajans Select Prompt" },
      },
    }

AI CLI INTEGRATION            *ajans.nvim-ajans.nvim-ai-cli-integration*

Ajans ships with a lightweight terminal wrapper so you can talk to local AI CLI
tools without leaving Neovim. Each tool runs in its own scratch terminal window
and shares helper prompts that bundle buffer context, current cursor position,
and diagnostics when requested.

API summary:

>lua
    require("ajans.cli").show(opts)
    require("ajans.cli").toggle(opts)
    require("ajans.cli").focus(opts)
    require("ajans.cli").hide(opts)
    require("ajans.cli").close(opts)
    require("ajans.cli").select(opts)
    require("ajans.cli").send(opts)
    require("ajans.cli").prompt(opts)
    require("ajans.cli").render(opts)

COMMANDS                                *ajans.nvim-ajans.nvim-commands*

Ajans provides a `:Ajans` command that wraps the Lua API.

>
    :Ajans <module> <command> [args]
<

Only the `cli` module is exposed.

CLI commands:

- `:Ajans cli show`
- `:Ajans cli toggle`
- `:Ajans cli hide`
- `:Ajans cli close`
- `:Ajans cli focus`
- `:Ajans cli select`
- `:Ajans cli send`
- `:Ajans cli prompt`

Example:

>
    :Ajans cli show name=claude focus=true
<

Lua equivalent:

>lua
    require("ajans.cli").show({ name = "claude", focus = true })
<

STATUSLINE INTEGRATION    *ajans.nvim-ajans.nvim-statusline-integration*

Use `require("ajans.status").cli()` to list active CLI sessions in your
statusline.

Example:

>lua
    table.insert(opts.sections.lualine_x, 2, {
      function()
        local status = require("ajans.status").cli()
        return " " .. (#status > 1 and #status or "")
      end,
      cond = function()
        return #require("ajans.status").cli() > 0
      end,
      color = function()
        return "Special"
      end,
    })

FAQ                                          *ajans.nvim-ajans.nvim-faq*

CLI tools not starting? ~

1. Verify the tool is installed: `which claude` (or your tool name).
2. Check `:checkhealth ajans` for tool installation status.
3. Try running the tool directly in your terminal.
4. Check `:messages` after attempting to start.

Terminal sessions not persisting? ~

Install tmux and enable the multiplexer:

>lua
    opts = {
      cli = {
        mux = {
          enabled = true,
        },
      },
    }
<

Do I need a GitHub Copilot subscription? ~

No. Ajans works with any supported AI CLI tool. You only need a Copilot
subscription if you choose to use Copilot CLI itself.

Will this work with Neovim 0.10? ~

No. Neovim >= 0.11.2 is required.

How do I add my own AI tool? ~

Add it to `cli.tools`:

>lua
    opts = {
      cli = {
        tools = {
          my_tool = {
            cmd = { "my-ai-cli", "--flag" },
          },
        },
      },
    }
<

How do I create custom prompts? ~

Add them to `cli.prompts`, then use `<leader>ap` or `:Ajans cli prompt`.
